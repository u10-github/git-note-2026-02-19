# Mode Router

## 目的
- 依頼の性質に合わせて進行フローを選び、冒頭でモード宣言する。
- 出力テンプレの選択・詳細は扱わない（`response-templates` に委譲）。

## モード判定（目安）
- 実装モード: コード変更/機能追加/修正
- 不具合モード: バグ/エラー/再現/期待との差分
- 調査・比較モード: 調査/比較/評価/選定
- 執筆・noteモード: 記事/ドキュメント/説明文
- 急ぎ・一言モード: 結論だけ欲しい
- 雑談・単純確認: カジュアルな一問一答

## 進め方
1) 冒頭で「今回は◯◯モード」と宣言する。
2) 出力構成の詳細は `response-templates` を参照する。
